Biblio on the go

Since the last 10 years or so the way we consume media has been completely changed. People can now stream music, film, read e-books... Not so long ago people were buying Cds, renting and returning film to the video store. Through these changes remained such institution like libraries pretty much the same and tried to diversify their offer. This led me to think. How might we join the convenience of the actual technology to a classical institution like the library?

The task to improve the overall experience of the library user is no easy endeavour as the offer of most public library is already very complete. I definitely had to dig in deeper to find ways of improvement and identify possible innovative solutions. In order to do that, I used different design thinking methods that I deemed relevant to explore in order to find possible solutions.

The problem

People are using less and less the library because there is more convenient and relatively cheap way of getting their media.

Objective

Goal

- Find out why are people not using the library more often

- Find out what are the hurdles

- Attract and keep new users
- Increase usage of the library by adding a new feature



Process

Problem statement reframing method

In order to rethink and get out of our traditional conception of the library, I used the following brainstorming method in order to expand the range of possibilities of action. The method is called the ‘’Problem Statement Reframing Method’’. The goal is to change a few word around a statement in order to broaden the audience, place and purpose of a given service. This exercise was done with a sample of 12 users. Here’s a sample of the findings.





A short description of your product or service.
A library is a place where the user can borrow books and study or read.

Description with keywords changed:
Version 1 (change the person/people who are involved)
A library is a place where people of all ages can borrow books and study or read.

Version 2 (change the setting where it happens)
The library system offers a platform where people of all ages can borrow books and study or read.

Version 3 (change the goal)
Goal
How can we improve the experience of the user in the library.

Underlying Goal
Stimulate the usership of the library.

Rewritten problem statement
How can we improve the experience of the user using the infrastructure of the library.

Problem definition

Not so long ago there was a time when people would go to a video store to rent movies and then watch them at home. The task was tedious and people had to do with what was available. One company came and revolutionized this business model. You are probably familiar with the streaming service that they now offer but actually Netflix started out as a mail order system. People would order Dvds and have them delivered home. They had a few weeks to watch them and would then conveniently return them in the mail. Such ideas stuck in my mind as to how to change the method to deliver content to the user. This inspired me to improve the experience in a public library.

This led me to think:

How might we bring the library to the user and not the other way around ?

Proposed User jouney

With all these information, I imagined the following system where the user could order books and have them delivered home.

I then came up with the idea of having a delivery system to improve the convenience of the public library while combining elements of online shopping / home delivery. In order to do so, I imagined a system that the library user could use to conveniently order books from the library and have them delivered to his home. Therefore, he wouldn’t have to go there during the opening hours and go through the stacks looking for his books. In exchange, he would pay a small fee similar to the one he would be paying to be using a used online bookstore. And since the delivery would be done in the city, it would be done with a bike messenger system to reduce to a maximum the C02 footprint. The user could then either drop his books at his nearby library using the overnight drop box or have a scheduled pickup.

Menu

One big element that was source to debate was wich navigation to use. There was the 2 different choices. A is at the bottom and B is on the top left corner. The users found in majority the top left navigation easier to use. This helped me to implement a better prototype.

During the test phase, the user found it easier to navigate with the top menu (menu B). Also, an additional factor is that the number of screens to navigate through is fairly small and the user don't need to switch often bewteen different screens.

Prototype

With the insight and first round of tests done with the wireframe, I was able to create a better prototype that I was confident to show to potential clients.

On the landing screen can the user either login or register.

The user is also informed from new arrivals, reminder etc. through notifications.


For a new user, he will be taken to the registration screen.

On the settings page can the user change his preferences.

The flyout menu allows the user to navigate quickly to the different screens.
The user can browse the catalog through the recommandations screen.

Or perform a search and then be taken to the search results screen.

In both cases, he can then click on any title and read the description. And then either order it or put it on his wish list.
On the borrowed items list can the user see his current list of items on loan.

He can also monitor the delivery status for a given order.

And also view his wish list and the current availability of the items.
